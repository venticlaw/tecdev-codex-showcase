AI methodology

The system is not just prompts. It is lanes, gates, waves, memory, and proof.

1. Channel Routing

Each brand has a dedicated lane. The Nucleus coordinates status and blockers, but it does not execute JW, JYW, or BOT work unless Josh explicitly reroutes it.

  • OPS owns Nucleus and cross-channel coordination.
  • JW owns technical authority, articles, site, and product assets.
  • JYW owns music products, catalog, offers, and checkout readiness.
  • BOT owns funding validation, drafts, and submission packets.

2. Approval Gates

Speed is allowed for local drafts and proofs. High-impact actions stop for human approval.

  • No DNS cutover without approval.
  • No payment, checkout, or public downloads without approval.
  • No Slack, email, forms, posts, or outreach without approval.
  • No Drive movement, deletion, backend storage, or public lead capture without approval.

3. Sub-Agent Waves

Meaningful work starts with a boundary pass, then splits into sharp function-named agents, then integrates the outputs.

  • Boundary pass: lane, scope, files, gates, blockers.
  • Wave 1: discovery/build slices like article, checkout, funding, or QA.
  • Wave 2: integrate, verify, update Nucleus, and record decisions.
  • Wave 3 only when extra QA, polish, or deployment readiness is needed.

4. Self-Improving Company OS

Every loop reads context, does bounded work, evaluates the output, updates memory, and surfaces the next approval or blocker.

  • Input: registry, runbook, backlog, source files.
  • Work: draft, validation note, proof page, status packet.
  • Evaluation: QA note, metric, failure, blocker.
  • Memory update: runbook, Nucleus todo, channel handoff.

Efficiency loops

Loop thinking turns every agent run into compound learning.

The operating pattern is simple: agents read the right context, do one bounded job, check the result, update memory, and surface the next approval or blocker. That keeps each run from starting cold and makes the system faster over time.

1 Input Registry, runbook, backlog, source files, approvals.
2 Action One local-safe task with a clear lane owner.
3 Output Draft, proof page, packet, validation note, or handoff.
4 Validation Link check, scan, status check, human review, or blocker.
5 Memory Nucleus update, runbook note, approval card, or next trigger.
Revenue Fastest money loop Rank the next offer, produce review-ready assets, surface one approval.
Content Authority loop Turn internal proof into scripts, articles, lead magnets, and demos.
Product Nucleus loop Improve the dashboard, approvals, status mirrors, and runbooks.
Learning Method loop Use mistakes, approvals, and launches to improve the next run.
